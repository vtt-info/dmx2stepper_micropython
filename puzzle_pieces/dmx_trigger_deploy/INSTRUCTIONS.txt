# DMX-Triggered Sensorless Homing - Deployment Instructions

## Overview
This package replaces the pushbutton trigger with DMX-based trigger for the sensorless homing function.

## Requirements
- RP2040 or RP2350 microcontroller (e.g., Raspberry Pi Pico, Pico 2, RP2040-Zero)
- TMC2209 stepper driver
- DMX512 signal source
- MicroPython v1.24.1 or later

## DMX Connection
- **DMX Input Pin:** GPIO 29 (connect DMX data line to GPIO 29)
- **Trigger Channel:** DMX Channel 1
- **Trigger Condition:** Value > 200 sustained for 2 seconds

## Files Included
- `main.py` - Boot entry point
- `example.py` - Main application (with DMX trigger logic)
- `stepper.py` - Stepper control with PIO
- `TMC_2209_driver.py` - TMC2209 driver
- `TMC_2209_uart.py` - TMC2209 UART communication
- `rgb_led.py` - RGB LED control

## Deployment Steps

### Option 1: Using mpremote (Recommended)

1. Connect your RP2040 to the computer via USB

2. Deploy all files:
   ```bash
   mpremote connect /dev/ttyACM0 fs cp *.py :
   ```
   (On Windows, use `COMx` instead of `/dev/ttyACM0`)

3. Run the code:
   ```bash
   mpremote connect /dev/ttyACM0 run main.py
   ```

### Option 2: Using Thonny

1. Open Thonny IDE
2. Connect to your RP2040
3. Upload all `.py` files to the device root
4. Run `main.py`

### Option 3: Manual Copy

1. Boot RP2040 into bootloader mode (hold BOOT button while pressing RESET)
2. Mount the device as a USB drive
3. Copy all `.py` files to the root
4. Press RESET to run

## Operation

1. **No DMX signal:** Displays "NO SIGNAL" if no DMX frames received for >1 second
2. **DMX signal present:** Monitors channel 1
3. **Trigger:** When CH1 > 200 for 2+ seconds, homing initiates automatically
4. **Homing cycle:** Steps motor to find endpoints, then centers between them

## Configuration

To change trigger settings, edit these variables in `example.py`:
- `dmx_trigger_channel = 1` - DMX channel to monitor (1-512)
- `dmx_trigger_threshold = 200` - Value above which triggers (0-255)
- `dmx_trigger_time_ms = 2000` - Hold time in milliseconds

## Troubleshooting

- **"Failed to start DMX receiver":** Check GPIO 29 wiring
- **"NO SIGNAL":** Verify DMX source is active and connected
- **"TMC driver UART does not react":** Check TMC2209 power and UART wiring
- **No response:** Hold GPIO 0 to GND on boot to skip auto-start

## Wiring Note

This version uses GPIO 29 for DMX input. Ensure your board has GPIO 29 available and properly connected to the DMX data line.
