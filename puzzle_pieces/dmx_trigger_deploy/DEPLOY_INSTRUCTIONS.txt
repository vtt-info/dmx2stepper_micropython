# DMX-Triggered Sensorless Homing - Deployment Instructions

## Overview
This package contains the modified code for sensorless stepper homing triggered by DMX512 signal instead of a pushbutton.

## Requirements
- Raspberry Pi Pico (RP2040) or Pico 2 (RP2350) running MicroPython v1.24.1+
- TMC2209 stepper driver
- DMX512 controller/sender

## Wiring
- DMX input: GPIO 29 (connect DMX signal to this pin)
- Stepper control: Same as original (see original wiring diagram)

## Files Included
- `main.py` - Boot entry point
- `example.py` - Main application (modified for DMX trigger)
- `stepper.py` - Stepper control class
- `TMC_2209_driver.py` - TMC2209 driver
- `TMC_2209_uart.py` - TMC2209 UART communication
- `rgb_led.py` - RGB LED control

## DMX Trigger Settings
- **DMX Channel:** 1
- **Trigger Value:** > 200
- **Hold Time:** 2 seconds

## Deployment Steps

### Method 1: Using mpremote (Recommended)

1. Connect your Raspberry Pi Pico to the computer via USB

2. Install mpremote (if not installed):
   ```bash
   pip install mpremote
   ```

3. Deploy files:
   ```bash
   mpremote connect /dev/ttyACM0 fs cp *.py :
   ```

4. Run the code:
   ```bash
   mpremote connect /dev/ttyACM0 run main.py
   ```

### Method 2: Using Thonny

1. Open Thonny IDE
2. Connect to your Pico
3. Upload all `.py` files to the Pico's root directory
4. Press Run or restart the Pico

### Method 3: Using rshell

```bash
rshell -p /dev/ttyACM0 cp *.py /pyboard/
```

## Verification

After deployment, connect to the serial terminal (115200 baud) and look for:
```
Initializing DMX receiver on GPIO 29...
DMX receiver started successfully
Waiting for DMX signal on channel 1 (value > 200 for 2 seconds)...
```

When DMX channel 1 exceeds 200 for 2+ seconds, the homing sequence will begin.

## Troubleshooting

- **"Failed to start DMX receiver"**: Check GPIO 29 wiring
- **"NO SIGNAL"**: No DMX signal detected - verify DMX controller is sending
- **Homing not triggering**: Ensure DMX channel 1 value is > 200 for 2 seconds
- **TMC driver not responding**: Check TMC2209 power and UART wiring

## Notes

- The pushbutton trigger (GPIO 9) has been removed
- If GPIO 0 is grounded on boot, auto-start is skipped (allows code editing)
- To revert to pushbutton, you would need to restore the original example.py
